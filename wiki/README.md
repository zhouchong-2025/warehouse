---

## 这套文档解决什么

把一个"修修补补很久"的半导体选型平台, 重构成**可扩展、可交接、可插件化**的系统。
核心病根: PDF 解析用"每格式一个脚本"的黑名单思路(16个extract脚本), 导致表头碎片化、数据错位。
核心药方: **坐标法内核 + 厂商配置**, 用 PDF 几何位置重建表格, 替代文本流猜测。

## 不可违背的铁律(贯穿所有文档)

1. **零假阳性** — 参数=PDF原文, 不做语义推断/翻译/猜测。
2. **系统性修复** — 发现bug先扫全量同类, 架构级修, 加测试, 不打补丁。
3. **mA≠A, ESD≠隔离** — 单位敏感, 不可混。
4. **参数乱码产品禁止自动打标签** — 标记人工审核。
5. **修复后必须审计+压测** — 用数据证明, 不接受口头保证。
6. **不要看到新格式就写 extract_v6/v7** — 扩展配置, 复用内核(本项目历史教训)。
