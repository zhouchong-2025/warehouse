## 0. 核心结论(先看这个)

实测四份 PDF 的真实结构后, 得出一个**反直觉但关键**的结论:

> **四份文档不是"四种格式各写一个解析器", 而是"一个坐标法内核 + 四套配置"。**
> 之前 16 个 extract 脚本的根本错误, 是把"配置差异"当成了"逻辑差异"。

四份文档底层都是**坐标对齐的表格**(除纳芯微部分手册页), 坐标法内核通用。
差异只在: 锚点是什么、表头是中/英文、是否单页、是否有手册介绍页混入。这些是**数据配置**, 不是代码分支。

## 1. 四份文档结构实测对比

| 维度 | 思瑞浦-模拟 | 思瑞浦-汽车 | 纳芯微 | 裕太 |
|------|-----------|-----------|--------|------|
| 页数 | 55 | 15 | **389** | **1** |
| 文档性质 | 选型册 | 选型册 | **产品手册+选型表混合** | 单页选型表 |
| 表头语言 | 英文 | **中文** | **中文** | **中文** |
| 表头行数 | 多行碎片 | 单行 | 多行(列名+单位行) | 单行 |
| 锚点(第一列) | Part Number/WPN | **"产品型号"** | **"产品型号"** | 无显式锚点(首列即型号) |
| section来源 | TOC + 页内标题 | 第一列"产品类别" | 每个选型表标题 | 顶部"系列"标签 |
| 表格分布 | 每section一表 | 单一大表跨页 | 145个表格页(分散) | 单页一张超宽表 |
| 特殊难点 | 表头碎片化 | 无(最规整) | **混入手册介绍页**, 需筛表格页 | 超宽表(列密集) |
| 当前数据质量 | 30%碎片(待P0修) | 0%(干净) | **34%乱(英文模板硬套)** | **列错位(CAN模板硬套)** |

### 关键洞察
- **思瑞浦-汽车/裕太当前"干净"或"错位", 都不是因为格式难, 而是因为有没有套对表头**。
  裕太 `28nm` 被塞进"供电电压", 是因为硬套了 CAN 收发器英文模板。**读取PDF里真实的中文表头就对了。**
- **纳芯微 34% 乱码的根因找到了**: page15 选型表里 `源边电阻:0.85 介电强度:5000 工作温度:-40~125`
  数据在 PDF 里**完全正确对齐**, 但旧数据用英文模板硬套, 错配成 `Status:0.85 Isolation:8 Supply:-40~125`。
  **不是 PDF 难解析, 是没用坐标法读真实表头。**

## 2. 统一架构: 一个内核 + 四套配置

### 2.1 内核(已实现, 即 extract_coord.py 的核心函数)
```
_spans()            通用: 取span+去噪
_build_columns()    通用: cx聚类 + 多行表头拼接
extract_table()     通用: 行分箱 + cx归位
行级section归属      通用
多重分类合并         通用
```
这些**一行不用改**, 四份文档共用。

### 2.2 每厂商配置(VendorProfile)
把差异收敛成一个配置对象, 而非代码分支:
```python
@dataclass
class VendorProfile:
    vendor_id: str                 # "3peak-analog" / "3peak-auto" / "novosense" / "yutai"
    anchors: set                   # 表头第一列锚点文本
    header_lang: str               # "en" / "zh"
    section_source: str            # "toc" / "first_col" / "table_title" / "top_label"
    leftmost_x: float              # PN列右边界(各PDF不同)
    col_tol: float = 8
    row_bin: float = 6
    skip_intro_pages: bool = False # 纳芯微需要筛掉手册介绍页
    noise_patterns: list = None    # 页眉页脚噪声
```

### 2.3 四套配置(实测得出)
```python
PROFILES = {
  "3peak-analog": VendorProfile(
      anchors={"Part Number","WPN"}, header_lang="en",
      section_source="toc", leftmost_x=170),            # 已验证

  "3peak-auto": VendorProfile(
      anchors={"产品型号"}, header_lang="zh",
      section_source="first_col",   # 第一列"产品类别"即section
      leftmost_x=170),               # 单一大表, 最规整

  "novosense": VendorProfile(
      anchors={"产品型号"}, header_lang="zh",
      section_source="table_title",  # 每个选型表标题(如"集成式电流传感器选型表")
      leftmost_x=90, skip_intro_pages=True),  # 需筛145表格页, 跳过手册介绍页

  "yutai": VendorProfile(
      anchors=set(),  # 无显式锚点, 首列即型号(YT开头)
      header_lang="zh", section_source="top_label",
      leftmost_x=80),  # 单页超宽表
}
```

## 3. 各厂商落地要点(执行模型按此实现)

### 3.1 思瑞浦-汽车(最简单, 先做)
- 中文表头单行, 不碎片 → `_build_columns` 直接出正确列名。
- section = 每行第一列"产品类别"(比较器/运放...), 不需要TOC。
- 锚点改为中文"产品型号", 表头列: 产品类别|产品型号|状态|封装|产品描述|可替代产品|应用领域。
- **物料归一关键**: 汽车型号(如 LM2901A-SO2R-S)的"可替代产品"列直接给出对应模拟料号(LM2901)
  → 这是物料主数据层 family 映射的现成数据源, 务必保留该列。

### 3.2 裕太(次简单)
- 单页超宽表, 中文表头单行。锚点: 首列 YT 开头型号(无"产品型号"字样, 用PN正则 `^YT\d`)。
- section: 顶部"80系列/车规"这类标签, 或全部归一个section。
- 列: 简介|封装|制程|状态|工作环境温度|端口|接口|Note。
- **验证点**: 制程列应是 `28nm/130nm`, 不是电压; 接口列是 `802.3bw`, 不是通道数。

### 3.3 纳芯微(最难, 最后做)
- **两步**: ①先筛出真正的选型表页(实测145个, 特征: ≥5对齐列且>40span且标题含"选型表")
  排除手册介绍页(产品介绍/性能/框图/应用场景的散文页)。
- ②对选型表页用坐标法, 中文多行表头(列名行+单位行, 如"介电强度"+"(Vrms)")。
- section = 选型表标题(如"集成式电流传感器选型表")。
- **务必重提取**: 现有 novosense 66款是英文模板硬套的废数据, 全删重来。实测 page15
  NSM2011 真实参数: 源边电阻0.85mΩ|介电强度5000Vrms|带宽240KHz|工作温度-40~125℃, 全部正确。
- **规模**: 389页可能产出远超66款, 注意 novosense 真实产品数。

## 4. 物料主数据层(MDM) — 跨厂商扩展的地基

P0 已实现单厂商内的"多重分类"(_sections数组)。跨厂商扩展需要 MDM:

### 4.1 三层结构
```
Family(型号族)         如 LM2901  ← 一颗芯片的逻辑身份
  └ Material(订货型号)  如 LM2901A-SO2R-S(汽车), LM2901(模拟)  ← 各厂册/各封装
      └ Source(来源)    哪份PDF/哪页/哪个section提取  ← 可追溯(铁律#1零假阳性)
```

### 4.2 数据来源(已找到)
- 思瑞浦-汽车的"可替代产品"列 = 汽车料↔模拟料的 family 映射现成数据。
- 同 family 跨册参数应一致(已验证模拟册内 TP2301 跨section参数一致)。
- 冲突时: 标记人工审核, 不自动合并(铁律#4)。

### 4.3 字段建议
```json
{
  "family": "LM2901",
  "materials": [
    {"pn":"LM2901","vendor":"3peak-analog","sections":["比较器"],"params":{...},"source":{"pdf":"...","page":15}},
    {"pn":"LM2901A-SO2R-S","vendor":"3peak-auto","sections":["比较器"],"grade":"车规","params":{...}}
  ]
}
```

## 5. 推广顺序(遵守"先验证核心case再推广"铁律)

```
P0  思瑞浦-模拟   ✅已验证, 待写库(见 P0_IMPLEMENTATION_GUIDE.md)
P1  思瑞浦-汽车   最简单, 中文单行表头, 验证"中文锚点+first_col section"
P2  裕太         单页超宽表, 验证"无锚点+top_label section"
P3  纳芯微        最难, 验证"筛表格页+中文多行表头", 全删重提取
P4  MDM物料层     四厂数据齐备后, 建 family 归一(用汽车册可替代列做种子)
```
每个 Pn 完成后: 列名总表交 FAE 审核 → 列级量纲校验 → 写库 → 审计压测。**不跳步, 不并行。**

### 5.1 三份文档的核心case验证结果(2026-06-08 实测)

P1/P2/P3 的**核心表已用坐标法跑通**(非仅结构探查), 证明"内核+配置"方向成立:

**思瑞浦-汽车** ✅ 7列全对齐:
```
产品类别:比较器 | 产品型号:LM2901A-SO2R-S | 状态:量产 | 封装:SOP14 |
产品描述:36V, General Comparator, BJT input, 4CH | 可替代产品:LM2901 | 应用领域:通用产品, xEV
```
→ "可替代产品:LM2901" 正确, 物料归一种子数据可用。

**裕太** ✅ 制程正确归位(不再串电压):
```
产品型号:YT8011A | 简介:GE PHY 车载1000base-T1 | 封装:QFN48 | 制程:28nm |
状态:MP | 工作环境温度:-40°C-125°C | 端口:802.3bw/802.3bp | 接口:RGMII & SGMII
```
→ 新发现边缘情况: ①首列表头是"80系列/车规"标签(非"产品型号") ②"接口"列有跨行值合并需处理。

**纳芯微** ✅ page15 电流传感器选型表 12列中文多行表头完美重建:
```
产品型号:NSM2011 | 源边电阻(mΩ):0.85 | 介电强度（Vrms）:5000 | 爬电距离(mm):8 |
工作电压(V):3.3/5 | 带宽(Hz):240K | 响应时间(us):2.2 | 工作温度(℃):-40~125 |
静态输出(V):50%VCC、10%VCC | 电流范围(A):0~30... | 封装:SOW16 | 应用等级:工规
```
→ 对比现数据库乱码 `Status:0.85|Isolation:8|Supply:-40~125`, **坐实34%乱码=英文模板硬套**。
→ 新发现边缘情况: "电流范围"列是跨多行的长值(需合并多行单元格)。

### ⚠️ 验证深度说明(诚实标注)
- **思瑞浦-模拟(P0)**: 全册71 section + 多边缘情况完整验证, 可写库。
- **P1/P2/P3**: 仅验证了**核心1-2个表/页**的提取, 证明方向成立。**全册推广验证待执行模型完成**:
  - 裕太: 跨行单元格合并(接口列)、是否有非PHY的异形行。
  - 纳芯微: 145个表格页是否有异形表(不同列数/无单位行/合并表头)、手册介绍页筛选准确率。
  - 汽车: 跨页延续、产品类别变化点。
- **不可假设三份和模拟册一样稳**, 每份推广时都要走完整 FAE 审核 + 量纲校验 + 压测。

### 关键: 不要为每份文档写新解析器
执行模型最容易犯的错(也是本项目历史教训): 看到新格式就写 extract_v6/v7。
**正确做法**: 扩展 `VendorProfile` 配置 + 复用坐标法内核。
若某厂商真的无法用配置覆盖(罕见), 才考虑内核加可配置 hook, 而非复制粘贴新脚本。

## 6. 验证标准(每厂商写库后)

复用 P0 的工具(validate_columns.py / compare_extraction.py), 标准一致:
- 碎片表头 <1%
- 列级量纲一致性: 各列值类型符合列名语义(电压列是数值, 温度列是区间...)
- 单位敏感: mA≠A, Vrms≠V(铁律#3)
- 乱码产品标记人工审核, 禁止自动打标签(铁律#4)
- 跑现有测试, 通过率不降
- 不确定项整理清单交 FAE, 不自行下"修好了"结论
