# Semantic Evidence Registry v1
# 单一真源：canonical semantic concept -> evidence phrases -> matching/audit policy
#
# 字段：
# - tag: canonical semantic concept
# - dimension: technology / feature / grade / spec / media
# - strength: must / nice / sort_hint / evidence_only
# - include: 适用品类（逗号分隔）
# - exclude: 排除品类（逗号分隔，可空）
# - fields: 证据字段（逗号分隔）
# - auto: precise regex 命中时能否自动修复数据层 evidence tag（供 audit 用）
# - aliases: matcher 用的显式证据短语；用 ;; 分隔，避免和 include/fields 的逗号冲突
# - regex: precise regex（审计/自动补 tag 用）
# - keywords: broad regex（coverage audit / phrase gap discovery 用）
# - query_regex: query_parser 用的触发 regex；仅放“用户会真的这么搜”的表达，不要把 detail 证据词全塞进来
#
# 说明：
# 1. aliases 供搜索运行时 matcher 使用；regex/keywords 供离线审计使用。
# 2. _features 仍保持结构性标签；semantic 命中来自 _params / _detail_* 证据。
# 3. 新概念先加 registry，再决定是否需要 parser 映射和 UI 解释。
