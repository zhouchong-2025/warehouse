### A. CAN SBC 误归 RS-485 收发器 (3peak-analog, 6款)

**证据 (跨vendor交叉验证 + features自相矛盾):**
- 3peak-analog 把这6款 section 标为 'RS-485 收发器', features 同时含矛盾标签 `SBC` 和 `RS-485`
- 3peak-auto 同型号 (TPT11695FQ-DFUR-S 等) section='SBC', 产品描述明确 "CAN SBC，5V/3.3V LDO, Watch dog, SPI, PN", 可替代产品=UJA1169 (NXP经典车规CAN SBC)
- params 的 Features 字段: "2 x Integrated LDO, Watchdog, SPI" — 这是 SBC 标志, 不是 RS-485 收发器
- 即: 这是集成 CAN 收发器的系统基础芯片(SBC), 与 RS-485 无关

**受影响 PN (3peak-analog):**
```
TPT11695XFQ, TPT11695XQ, TPT11695FQ, TPT11695Q
TPT11693FQ, TPT11693Q
```

**建议修法:**
1. _section: 'RS-485 收发器' → 'SBC'
2. _features: 删除 `RS485收发器`、`RS-485` 标签; 保留/补充 `SBC`、`CAN-FD`、`车规AEC-Q100`、`1通道`、`8Mbps`(数据速率真实值)
3. 在 autofix.py PREFIX_SECTION 加 `'TPT11693': 'SBC'`、`'TPT11695': 'SBC'` (注意: 仅这两个型号族, 不要用更短前缀以免误伤其他 TPT116xx)
4. 确认 SBC↔RS-485 是否该进协议互斥对: **不要简单互斥** — SBC 是复合芯片可集成 RS-485。这里靠 PREFIX_SECTION 精确改 section, 由 section→tag 自然清理, 不靠互斥护栏

**验证:** 前端搜 "RS-485" 不应出现 TPT11693/11695; 搜 "CAN SBC" 或 "SBC" 应出现; 搜 "UJA1169 替代" 应反查命中

## P1 — 纳芯微规格标签系统性漏提取 (537款中326款, 61%)
═══════════════════════════════════════════════════════════

**问题性质:** 不是误标, 是 tag_config.py 的 CATEGORY_PARAMS 没有为纳芯微这些品类配置「参数列→标签」映射。
导致 326款 _features 只有 section 名复读 + 等级词 (如 "集成式电流传感器选型表 工业级"), 无任何可搜索/可排序的规格标签。
后果: 这些产品无法被规格搜索命中 (搜"5000Vrms电流传感器""宽带"召不回), 无法按规格排序。

**证据样本 (NSM2011, 集成式电流传感器):**
- 当前 _features: `集成式电流传感器选型表 工业级`
- params 实际含丰富规格: 源边电阻0.85mΩ | 介电强度5000Vrms | 爬电距离8mm | 工作电压3.3/5V | 带宽240KHz | 响应时间2.2us | 电流范围±20~±50A | 封装SOW16
- 这些全部可提取成标签, 但一个都没提

**按品类漏提取统计 (漏/总):**

**建议修法 (codex 执行, 按品类优先级):**
1. 在 tag_config.py 的 CATEGORY_PARAMS 为纳芯微品类补「参数列→extract函数」映射。优先高价值可搜品类:
   - 集成式电流传感器/线性电流传感器: 电流范围(A)→Iout式标签或电流量程标签, 介电强度(Vrms)→隔离标签, 带宽(Hz)→带宽标签
   - 温度传感器: 精度、量程
   - LDO/DC-DC: Vin/Vout/Iout (复用现有 extract_vin/vout/iout)
   - CAN/LIN/隔离CAN收发器: data rate→speed标签(注意已改单一真实值)
   - ADC: 分辨率bit、采样率
2. **铁律**: 纳芯微参数列名是中文 (源边电阻/介电强度/带宽/电流范围), 与思瑞浦英文列名不同, 需新写中文列名匹配。注意单位: 介电强度是 Vrms, 带宽是 Hz(注意K/M), 电流范围是 A。
3. **不要语义推断**: 参数缺失或乱码(如"电流范围(A) -20~20、-30~30、"这种带顿号的多值)不强行解析, 标记人工审核。
4. 参数值含多个候选(如电流范围"±30、30、±50、50")属于选型表一行覆盖多个子型号, 提取时取最大量程或标记, 不要瞎拆。

**验证:** 补映射后跑 autofix.py, 确认这些品类 _features 出现真实规格标签; 前端按规格搜索/排序应能命中纳芯微产品。

### P3-1. 电流/电压标签梯子展开重构 (最高价值, 同速率的病根)

**问题:** VIN/VOUT/IOUT 标签在 autofix.py + tag_config.py 两处都是「梯子展开」
(if 真实值 >= 档位: 打上该档标签), 与已修复的速率标签同一个病根。后果同速率:
①真实值不在标准档位上就丢失 ②亚单位真实值无标签 ③一颗料挂一堆它并非的低档标签。

**已验证的修法 (照速率试点这套, 已跑通可复用):**
1. autofix.py: VIN/VOUT/IOUT 三个块的梯子展开 → 单一真实值标签
   (参考已改的 speed 块: 行~256, `parts.append(rstr)` 单值)
2. tag_config.py: extract_vin/extract_vout/extract_iout 三个函数 → 单一真实值
   (参考已改的 extract_speed: 行~44, 直接 return [真实值标签])
3. query_parser.ts: PARAM_RULES 的 V/A 规则 → 发单一真实值标签 (参考已改的 bps 规则用 fmtNum)
   DOWNGRADABLE_FAMILIES 已含 'Vin'/'Iout'? 确认 Vout 是否需加 (要N V/A, 产品≥N满足)
4. constraint-match.ts: tagSatisfied 加 Vin/Vout/Iout 的 downgradable 数值≥比较分支
   (参考已加的 Mbps 分支: 行~85, 解析标签数值取max与要求值比较)

**铁律提醒:**
- 单位检测铁律 (见 MEMORY): IOUT 必须检测单位 mA vs A。params含"Iout Max (mA)"→值/1000;
  含"Iout Max (A)"→直接用。LDO值>100大概率是mA。不要梯子改单值时把这个单位逻辑弄丢。
- 向下兼容语义: 要 1A 的, 产品 6A 也满足 (≥比较)。要 5V Vin 的, 36V 输入也满足。

**验证:** 改后跑 autofix→test_all→build→test_constraint_layer 全绿;
数据核实真实值恢复(非标准档位的电流/电压值不再丢)、亚单位值(0.5A/0.x V)有标签;
前端搜 "1A LDO" / "5V 输出" / "36V 输入" 召回正确, "高电流DCDC" 按 Iout 降序排序正确。
